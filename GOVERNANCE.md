# Governance and Community Structure

This document describes how TruShell is governed, how decisions are made, and how contributors can participate in leadership.

Mission
- Make TruShell the most trusted, performant, and compatible terminal shell for Linux users.

Structure
- Steering Committee: small group (3-7) responsible for high-level strategy, legal/finance decisions, and overall direction.
- Maintainers/Core Team: individuals responsible for core subsystems (runtime, PTY, shell parser, packaging, security).
- Working Groups: cross-cutting teams that anyone can join — Runtime WG, Extensions WG, UX WG, Docs WG, Packaging WG.

RFC Process
- Proposals are made as RFCs in /docs/rfcs or GitHub Discussions.
- Discussion window: 2 weeks by default.
- Approval: majority of maintainers + one steering member for major changes.

Trusted Committer criteria
- 10 merged PRs that follow project standards
- Demonstrated helpful code review behavior
- One sponsor from the core team and passing a short vetting call

Decision escalation
- If maintainers disagree, escalate to Steering Committee for arbitration.

Security & reporting
- Responsible disclosure: email security@trufoundation.org
- Security issues are triaged by the security maintainer; public disclosure follows coordinated timeline.

This is a living document and may be updated by the Steering Committee.
